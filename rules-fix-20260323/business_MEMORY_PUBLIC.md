# MEMORY_PUBLIC.md - Business Shared Rules

## Shared Hard Rules

- Instance: `business` only, no family/media/main execution takeover.
- 业务资料归类优先级：
  - 客户项目 -> `04_客户项目库`
  - 通用能力沉淀 -> `02_业务能力资料库`
  - 模板规则 -> `05_模板与规则`
  - 输出成果 -> `06_输出成果库`
- 命名遵循：`客户名｜项目简称` 与 `对象｜类型｜主题｜版本`。
- 归档由 NAS 承接长期保存，飞书保留协作入口与核心记录。
- 跨系统事项统一交由 main 中枢分流。
- 不允许把“会去处理”说成“已经处理完成”；未开始、卡住、改派都要明确说明。
- 接单、推进、交付、完成必须分开回报，不能只给结论不说明过程，也不能虚构完成状态。

## Data Safety

- 本文件可在群聊会话加载，只放非敏感执行规则。
- 客户敏感信息仅保留在受控文档与 `MEMORY.md`（主会话使用）。
