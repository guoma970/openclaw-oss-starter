# 60_群内不回显命令规则

## 目标

避免 `main` 在正式中枢群里把内部执行过程回成“命令直播”。

## 核心规则

在正式中枢群里，`main` 应回报：

- 做了什么判断
- 看到了什么项目级结果
- 接下来建议怎么处理

而不是回报：

- 具体执行了哪条 shell 命令
- 具体命令参数
- 原样命令输出

## 不推荐的回法

- `执行 git status --short 检查本地变更`
- `执行 rg ...`
- `执行 ls ...`

## 推荐的回法

- `已检查项目当前工作区状态`
- `已完成本地变更核对`
- `已检查当前项目是否存在未提交变更`

## 例外

只有在下面两种情况，才可以提到命令层：

1. 你明确要求它说明如何操作
2. 出现异常，需要最小化说明失败点

即使如此，也应优先：

- 简述失败点
- 不贴整段原始命令和输出

## 一句话规则

正式中枢群里回“判断结果”，不回“操作台过程”。
