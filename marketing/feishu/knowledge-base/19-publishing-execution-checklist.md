# 对外发布执行清单

## 目的

把项目相关的对外发布动作统一成一份 checklist，避免不同平台漏项。

## 适用范围

- GitHub 仓库更新
- ClawHub skill 发布或版本同步
- 飞书知识库同步
- Feishu 群内同步
- X / Twitter 短帖
- 小红书发布改写

## 标准流程

### 1. 本地确认

- 是否已完成本地仓库修改
- 是否已通过校验
- 是否已提交并 push
- 是否已明确版本或更新主题

### 2. GitHub

- 仓库首页是否需要同步
- skill 列表或版本号是否需要同步
- changelog / release note 是否需要同步
- 相关 issue / roadmap 是否需要同步

### 3. ClawHub

- 是否需要新发布
- 是否需要版本 bump
- skill 描述是否需要调整
- 是否需要再次检查可见状态

### 4. 飞书知识库

- 项目总览是否需要同步
- 对外发布文案页是否需要同步
- 申请与介绍素材是否需要同步
- 其他相关规则页或复盘页是否需要同步

### 5. 群内 / 协同同步

- 是否需要发到 `media` 对应群
- 是否需要补阶段成果汇报
- 是否需要把链接回填到知识库

### 6. 对外营销文案

- 是否需要 Feishu 短版
- 是否需要 X 英文短帖
- 是否需要小红书改写稿
- 是否需要公众号 / 长文引子

### 7. 国内平台执行

- 是否需要 `果爸` 承接小红书发布
- 是否已有对应改写稿
- 是否要在发布后补复盘记录

## 推荐回报格式

- 状态：
- 已完成平台：
- 待完成平台：
- 链接：
- 下一步：

## 最小执行版本

如果这次只是一次轻量更新，至少确认：

- GitHub 是否已同步
- 飞书知识库是否已同步
- 是否需要 ClawHub
- 是否需要一条对外短文案
