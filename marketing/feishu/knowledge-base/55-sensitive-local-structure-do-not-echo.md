# 55_敏感本地结构禁止回显规则

## 目标

明确哪些本地结构即使在内部协作群里，也不应默认直接回显。

## 禁止默认回显的内容

### 1. 本地实例根目录

- `01-main`
- `02-family`
- `03-media`
- `04-business`
- `05-ztl970`

### 2. `.openclaw` 结构

- `.openclaw/`
- workspace-builder 内部结构
- agent / runtime / plugin 私有目录

### 3. 业务或家庭内部目录名

- 家庭规则目录
- 业务客户目录
- 私人素材目录
- 仍带真实业务语义的内部文件夹

## 为什么

这些信息虽然不一定直接进入公开仓库，但仍然属于：

- 私有运行结构
- 内部协作结构
- 个人或家庭侧信息线索

它们不适合在群里被原样展开。

## 推荐替代表达

不要回：

- `Repository: /Users/ztl/01-main`
- `?? 07_家庭规则与提醒/`

改成：

- `main 实例存在未提交变更`
- `家庭相关实例存在未公开的本地结构变化`
- `业务相关实例存在内部目录调整`

## 一句话规则

可以回“哪类项目有变化”，不要回“本地目录树长什么样”。
