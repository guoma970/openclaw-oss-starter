# 可持续发布指令清单

## 目的

这一页用于整理适合持续运行的发布型任务，让 `main` 可以长期调度。

## 适合持续执行的任务

### 1. 日常同步检查

目标：

- 检查 GitHub、ClawHub、飞书知识库是否存在不同步
- 如有差异，由 `main` 派发给 `media`

建议节奏：

- 每天一次

### 2. 周度项目复盘

目标：

- 复盘本周项目更新
- 检查首页、技能页、知识库页、发布文案是否一致
- 整理下一步待办

建议节奏：

- 每周一次

### 3. 通用指令沉淀

目标：

- 从近期工作中提炼稳定的通用指令
- 更新到通用指令库
- 判断哪些内容值得公开发布

建议节奏：

- 每周一次

### 4. 新 skill 发布流程

目标：

- 对新 skill 做结构检查
- 完成仓库说明更新
- 如满足条件，继续同步 GitHub、ClawHub、飞书知识库

触发方式：

- 有新 skill 时手动触发

### 5. 对外文案刷新

目标：

- 更新项目简介、短介绍、群发文案、平台适配文案
- 保持 GitHub、飞书知识库、营销文案一致

建议节奏：

- 每周一次或有版本更新时触发

## 推荐的长期工作流

1. `main` 接收聊天指令或自动化检查
2. `main` 判断是否需要代码改动、本地文档改动或公开同步
3. 如需公开同步，派发给 `media`
4. `media` 负责 GitHub 展示、ClawHub、飞书知识库和营销文案同步
5. `main` 返回结果并更新复盘记录

## 建议优先级

- 高优先级：
  - 日常同步检查
  - 周度项目复盘
- 中优先级：
  - 通用指令沉淀
  - 对外文案刷新
- 按需触发：
  - 新 skill 发布流程

## 适合持续维护的输出

- GitHub 首页
- ClawHub 版本说明
- 飞书知识库项目目录
- 对外发布文案
- 通用指令库
