# Public-safe Skill 规则

## 目标

确保所有进入公开链路的 skill 都是：

- 通用的
- 可复用的
- public-safe 的
- 不包含个人或家庭私密信息

## 核心判断

一个 skill 只有同时满足下面两点，才适合公开发布：

1. 能被其他人复用
2. 不暴露个人、家庭、账号或私密运行态信息

## 可以公开发布的内容

- 通用任务流程
- 通用提醒与检查逻辑
- 通用角色分工模板
- 通用知识库整理方法
- 通用发布与营销工作流
- 通用本地部署方式
- 通用 skill 模板与说明

## 不应公开发布的内容

- 真实姓名
- 家庭成员身份细节
- 学校名称
- 群 ID、账号 ID、会话代码
- 私人作息和精确时间安排
- 私有记忆、私有运行态、heartbeat 类内容
- 只适用于你个人家庭、无法泛化的内部规则
- 能识别未成年人身份的信息

## 抽象原则

如果内容原本来自个人场景，公开前要先抽象成：

- 通用角色
- 通用流程
- 通用边界
- 通用示例

而不是直接保留原始人物、原始账号、原始背景。

## 一句话规则

能抽象成“别人也能复用的模板”才发布；
只能服务个人或家庭本身的内容，不发布。
