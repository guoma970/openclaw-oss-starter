# 发布决策模板

## 目的

让 `main` 在做发布前判断时，输出统一、清晰、可执行的结论。

## 标准结构

### 1. 结论

- `可公开发布`
- `仅进入知识库`
- `仅本地保留`
- `先抽象再发布`

### 2. 分级

- `A`
- `B`
- `C`
- `D`

### 3. 理由

简短说明判断依据，优先覆盖：

- 通用性
- public-safe
- 结构完整性
- 是否命中禁止清单

### 4. 建议动作

#### 若为 A

- 进入 GitHub / ClawHub / 飞书公开链路

#### 若为 B

- 进入飞书知识库
- 不做公开 skill 发布

#### 若为 C

- 仅保留本地
- 不进入任何公开链路

#### 若为 D

- 先做抽象化改写
- 先补结构
- 再重新检查

## 推荐回报模板

```text
状态: done
结论: 可公开发布 / 仅进入知识库 / 仅本地保留 / 先抽象再发布
分级: A / B / C / D
理由:
- ...
- ...
动作:
- ...
下一步:
- ...
```

## 示例 1：可公开发布

```text
状态: done
结论: 可公开发布
分级: A
理由:
- 该 skill 已抽象为通用模板
- 未发现个人或账号痕迹
- 结构完整
动作:
- 可进入 GitHub / ClawHub / 飞书公开链路
下一步:
- 同步公开侧说明和发布文案
```

## 示例 2：仅知识库沉淀

```text
状态: done
结论: 仅进入知识库
分级: B
理由:
- 内容有长期价值
- 但更适合内部方法论沉淀，不适合做成公开 skill
动作:
- 同步到飞书知识库
下一步:
- 如后续可抽象，再重新评估是否公开
```

## 示例 3：仅本地保留

```text
状态: done
结论: 仅本地保留
分级: C
理由:
- 内容包含私人运行态和账号痕迹
- 不适合进入公开链路
动作:
- 仅保留本地
下一步:
- 如需公开，先彻底抽象和清理
```

## 示例 4：先抽象再发布

```text
状态: done
结论: 先抽象再发布
分级: D
理由:
- 方向有公开价值
- 当前仍带个人场景痕迹
- 结构也不完整
动作:
- 先抽象示例和边界
- 先补齐结构文件
下一步:
- 完成后重新走发布前检查
```
