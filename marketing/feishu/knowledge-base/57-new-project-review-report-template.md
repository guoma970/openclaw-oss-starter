# 57_新项目审核标准回报模板

## 目标

让 `main` 在审核新项目后，能给你一个稳定、可执行、可拍板的回报。

## 推荐模板

```text
状态: done
项目: <项目名>
项目类型判断: 公开项目 / 内部推进项目 / 商业化敏感项目
建议分级: A / B / C / D
同步范围判断: GitHub / 飞书知识库 / 仅本地 / 待确认
理由:
- ...
- ...
建议动作:
- ...
- ...
是否需要你拍板: 是 / 否
下一步:
- ...
```

## 推荐结论类型

### 1. 建议进入公开主线

适合：

- 通用
- public-safe
- 有对外价值

### 2. 建议只进入知识库

适合：

- 有内部沉淀价值
- 暂不适合公开为项目或 skill

### 3. 建议只保留本地

适合：

- 明显私人
- 商业化敏感
- 尚未抽象

### 4. 建议先抽象再判断

适合：

- 方向有价值
- 但当前边界和表达还不适合公开

## 示例

```text
状态: done
项目: 羲果陪伴
项目类型判断: 商业化敏感项目
建议分级: C
同步范围判断: 仅本地保留
理由:
- 与未来商业化路径强相关
- 当前不适合默认公开
建议动作:
- 先保留在本地与内部沉淀
- 如后续要公开，先做专门抽象
是否需要你拍板: 是
下一步:
- 请确认是否将其长期保持为商业化敏感项目
```
