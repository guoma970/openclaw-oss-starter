# 发布前检查清单

## 目标

在任何 skill 进入 GitHub、ClawHub 或飞书公开知识库前，先做统一检查。

## 第一层：通用性检查

- 这个 skill 是否是通用模板，而不是私人专属规则
- 这个 skill 是否可以被其他人理解并复用
- 标题和描述是否仍然带有明显个人场景痕迹
- 示例是否已经抽象成通用角色和通用流程

## 第二层：隐私与 public-safe 检查

- 是否包含真实姓名
- 是否包含学校名称
- 是否包含群 ID / 账号 ID / 会话代码
- 是否包含私人作息或精确家庭安排
- 是否包含未成年人可识别信息
- 是否包含私有记忆、私有运行态或 heartbeat 内容

## 第三层：结构检查

- 是否存在 `SKILL.md`
- 是否存在 `README.md`
- 是否存在 `agents/openai.yaml`
- 是否有必要的 examples 或 release notes
- 描述是否足够清楚

## 第四层：公开链路检查

- GitHub 首页是否需要同步
- ClawHub 是否需要发布或 bump 版本
- 飞书知识库是否需要新增或更新说明页
- 是否需要补一条对外发布文案

## 结果判断

### 可以发布

满足条件：

- 通用性通过
- public-safe 通过
- 结构完整

动作：

- 允许进入 GitHub / ClawHub / 飞书公开链路

### 暂不发布

满足任一情况：

- 仍然带有明显个人痕迹
- 缺少关键结构文件
- 有私密信息风险

动作：

- 只保留本地
- 先抽象、清理或补结构

## 推荐回报格式

```text
状态: done / blocker
动作: 已完成发布前检查
证据: 通用性 / public-safe / 结构检查结果
下一步: 可公开发布，或先本地清理后再发
```
