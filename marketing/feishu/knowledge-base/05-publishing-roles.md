# 发布执行规则

## 分工原则

### media

- 负责 AI 可辅助或可自动化的平台内容整理
- 负责项目动态、发布文案、阶段同步
- 负责把对外内容先沉淀到飞书知识库
- `media` 是本地实例名 / 目录名
- 飞书里对应的机器人名字叫 `小果`

### 果爸

- 负责国内平台的小红书实际发布
- 承接账号 `陆壹儿童节` 的执行动作
- 根据知识库中的发布文案做最终落地

## 当前项目规则

- 项目知识库归属到 `自媒体空间`
- 当前项目目录为 `11_OpenClaw开源项目`
- 标准页面包括：
  - `01_项目总览`
  - `02_对外发布文案`
  - `03_申请与介绍素材`
- 新增执行规则页与账号矩阵补充页可作为后续扩展页

## 协同顺序

1. 先由 `media` 整理文案和项目说明
2. 再由 `media` 实例通过飞书机器人 `小果` 沉淀到知识库
3. 最后由 `果爸` 负责国内平台实际发布
