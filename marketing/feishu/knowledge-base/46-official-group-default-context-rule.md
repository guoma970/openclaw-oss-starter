# 46_正式中枢群默认上下文规则

## 目的

让正式中枢群里的日常短指令不需要每次重复说明：

- 当前项目
- 当前仓库
- 当前协作链

## 默认上下文

在正式中枢群中，若未明确说明其他项目，则默认指向：

- 项目：`openclaw-oss-starter`
- 本地工作区：`/Users/ztl/Documents/New project`
- GitHub 仓库：`guoma970/openclaw-oss-starter`
- 公开链路：GitHub / ClawHub / 飞书知识库

## 默认协作对象

在正式中枢群中，若未明确指定其他角色，则默认按以下协作链理解：

- `main`：入口、判断、分流、回报
- `小果`：公开同步、知识库同步、文案整理
- `果爸`：国内平台发布执行

## 直接影响

因此，像下面这种短句应被视为上下文充分：

- `请从 main 看今天哪些值得更新 GitHub。`
- `请从 main 判断这次要不要同步 ClawHub。`
- `请从 main 同步知识库。`

## 只有这些情况才需要再追问

- 明确提到其他仓库
- 明确提到其他工作区
- 明确提到其他项目
- 任务过于模糊，可能会改错目标

## 一句话规则

正式中枢群默认继承当前项目上下文，短口令应优先成立，不要轻易因为缺少 repo 重复追问。
