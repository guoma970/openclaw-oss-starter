# 54_群内回报脱敏规则

## 目的

避免 `main` 或 `小果` 在飞书群里直接回传不必要的本地结构信息。

## 核心原则

飞书群里的回报应默认脱敏。

能不写真实路径，就不要写真实路径。  
能不写真实目录名，就不要写真实目录名。

## 群里不应直接回的内容

- 本机绝对路径
- 用户主目录结构
- `.openclaw/` 路径
- 家庭 / 业务 / 媒体 / 私人实例目录名
- 未脱敏的仓库扫描结果
- 大段 `git status` 原样输出

## 群里推荐怎么回

### 可以这样写

- `默认项目工作区`
- `已登记的内部项目`
- `商业化敏感项目`
- `存在未提交变更`
- `存在待抽象内容`

### 不建议这样写

- `/Users/ztl/01-main`
- `/Users/ztl/03-media`
- `.openclaw/`
- `00_总入口与说明/`
- `07_家庭规则与提醒/`

## 路径回报规则

如果确实需要引用路径：

- 优先用项目 key
- 优先用相对概念
- 必要时只在本地日志或私有知识库里保留真实值

## 一句话规则

飞书群回报要以“项目级结论”为主，不以“本地目录扫描明细”为主。
