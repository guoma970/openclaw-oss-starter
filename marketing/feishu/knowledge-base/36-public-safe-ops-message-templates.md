# Public-safe 运营消息模板

## 目标

把原始的运营群消息模板抽象成一版可复用、可公开、public-safe 的通用模板。

适用场景：

- 内容运营协同
- 任务推进与回报
- 素材生产流程
- 发布确认与复盘
- 可供 OpenClaw 或类似系统做结构化提取

## 设计原则

- 不绑定具体账号名
- 不绑定具体群名
- 不绑定具体业务体系
- 保留可结构化提取的关键字段

## 1. 选题模板

```text
【消息类型】选题
【主归属】content / ops / business
【协同归属】无 / content / ops / business
【任务ID】
【主题】
【目标】本次选题要解决什么问题
【受众】给谁看
【内容角度】核心表达角度
【输入资料】已有素材 / 链接 / 文档
【输出要求】要形成什么内容
【当前状态】pending / ready / doing
【负责人】
【截止时间】
【下一动作】
【回报要求】是否需要给出提纲 / 脚本 / 标题方案
```

## 使用要求

- 模板字段可以补充，但不要删除核心字段
- 同一任务在不同阶段流转时，应沿用同一个 `任务ID`
- 如果某个项目需要更严格的主归属 / 协同归属规则，应在项目内部规则页单独说明

## 公开定位

这是一份可公开的通用协同模板，不对应任何具体账号、群、家庭或业务体系。
