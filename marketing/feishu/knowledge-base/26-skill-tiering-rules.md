# 技能分级规则

## 目标

对所有 skill 或候选流程做统一分级，避免“不是不能用，但不适合公开”的内容误入公开链路。

## 分级结果

### A 级：可公开发布

满足条件：

- 通用
- public-safe
- 可复用
- 结构完整
- 适合进入 GitHub / ClawHub / 飞书公开知识库

适合内容：

- 通用 skill
- 通用部署模板
- 通用知识库与发布流程
- 通用协同规则

处理动作：

- 可进入完整公开链路

### B 级：仅知识库沉淀

满足条件：

- 有价值
- 可沉淀
- 但不适合做成对外公开 skill

适合内容：

- 项目内部方法论
- 发布节奏、复盘、执行规则
- 账号矩阵
- 回报模板、异常规则

处理动作：

- 进入飞书知识库
- 不进入 ClawHub
- 一般不作为公开 skill 发布

### C 级：仅本地保留

满足任一情况：

- 带个人、家庭或账号痕迹
- 私有运行态内容
- 无法抽象成通用模板
- 存在隐私风险

适合内容：

- 私有记忆
- 私人会话代码
- 私有 heartbeat / 运行态
- 家庭专属细则
- 无法泛化的内部规则

处理动作：

- 只保留本地
- 不进入 GitHub / ClawHub / 飞书公开知识库

### D 级：先抽象再判断

满足条件：

- 有潜在公开价值
- 但当前仍带个人场景或结构不完整

适合内容：

- 从个人场景提炼出来但还没抽象好的流程
- 示例还带身份痕迹的 skill
- 结构不完整的候选 skill

处理动作：

- 先做抽象化改写
- 先补结构
- 再重新走发布前检查

## 分级判断顺序

1. 先看是否命中“不应公开内容清单”
2. 再看是否具备通用性
3. 再看是否结构完整
4. 最后判断进入 A / B / C / D 哪一级

## 一句话规则

- A：公开发布
- B：知识库沉淀
- C：仅本地保留
- D：先抽象再判断
