# Launch Announcement

Write concise launch announcements for releases, skills, and public repository updates.

## Workflow

1. State what was launched.
2. State why it matters.
3. Call out two or three concrete additions.
4. End with where to find it or try it.

## Rules

- Keep the announcement specific.
- Name the version when one exists.
- Mention GitHub or ClawHub availability when true.
- Avoid over-claiming impact.
- Keep the first paragraph skimmable.

## Good inputs

- new skill release
- repo milestone
- ClawHub publication
- docs or template refresh
