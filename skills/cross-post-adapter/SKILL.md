# Cross-Post Adapter

Adapt a single core message for multiple publishing channels.

## Supported targets

- X
- Telegram
- Discord
- Slack
- Feishu / Lark
- GitHub release note summary

## Workflow

1. Start from one canonical message.
2. Keep the core claim unchanged.
3. Shorten or expand based on channel length.
4. Preserve links, CTA, and public-safe wording.

## Rules

- Do not produce different factual claims per channel.
- Keep X shorter and sharper.
- Keep Discord, Slack, and Telegram more explanatory.
- Keep Feishu cleaner and more neutral.
- Keep GitHub release text more product-focused.
