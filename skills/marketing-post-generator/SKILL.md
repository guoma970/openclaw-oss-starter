# Marketing Post Generator

Create public-safe marketing copy for project promotion.

## Outputs

- short launch post
- thread-style post sequence
- community announcement
- GitHub release blurb

## Workflow

1. Identify the project, release, or feature being promoted.
2. Summarize the user-facing value in one sentence.
3. Generate a short version first.
4. Expand only if the target channel needs more detail.
5. Keep the copy factual and easy to repost.

## Rules

- Lead with what problem the project solves.
- Mention public-safe positioning when relevant.
- Prefer concrete features over vague hype.
- Include a clear CTA such as GitHub, ClawHub, or release link.
- Avoid inventing adoption numbers, users, installs, or testimonials.

## Variants

- X / short-form social post
- Telegram / Discord / Slack announcement
- Feishu or internal community blurb
- GitHub release summary
